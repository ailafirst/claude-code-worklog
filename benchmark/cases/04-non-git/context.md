今天没写代码，在 `design/auth-v2.md` 起草新的鉴权方案。核心是一个二选一的权衡：

- **JWT（无状态）**：吸引力在于服务端不用存会话、横向扩展省事。
- **不透明 session token + 服务端会话存储（有状态）**：每次校验要查一次存储。

**倾向后者**，决定性理由是**吊销**：管理后台一旦某个 admin token 泄漏，必须能**立刻失效**。
JWT 在过期前无法即时吊销（要么维护黑名单——那就又变回有状态、白拿了无状态的复杂度）。
对内部系统而言，admin 被盗的即时吊销能力 > JWT 的无状态便利。

**仍未解决的开放问题**：移动端有**离线窗口**——离线期间拿什么校验？如果走有状态 session，
离线就没法查存储。可能要给移动端发一个短时效的派生凭证，但这会重新引入"短期内无法吊销"的
口子，跟选有状态的初衷打架。这条没定，留着下次。
